## 1. 임베딩/리랭킹 모델 교체 ✅

**코드 수정 완료.** 사용 방법:

```bash
# 기존 BAAI 모델로 빌드 (기본값)
python src/services/corpus_loader.py --table all --embed-model baai

# dragonkue 한국어 모델로 빌드
python src/services/corpus_loader.py --table all --embed-model dragonkue
```

서빙 모델 전환은 `config.py` 상단 한 줄만 변경:

```python
_EMBED_MODEL_VARIANT = "dragonkue"  # "baai" ↔ "dragonkue"
```

**중요**: corpus_loader의 `--embed-model`과 config.py의 `_EMBED_MODEL_VARIANT`는 반드시 같은 값이어야 한다. DB를 `dragonkue`로 빌드했으면 서빙도 `dragonkue`여야 한다.

## 2. designs 청킹 개선

### 현황

현재 `load_designs()`는 페이지 단위로 청킹하고, 각 페이지에서 섹션 번호를 추출한다. KOSHA Guide 문서 구조(`제N장 → N. → N.N → (N)`)에서 섹션이 페이지 경계에 걸리면 맥락이 끊긴다.

### 테스트 방향

**목표**: 섹션 단위 청킹이 실제로 검색 품질을 높이는지 확인.

1. **현행 DB로 기준 점수 확보**
   - `ragas/` 파이프라인에서 design 관련 질문만 추려서 RAGAS 점수(faithfulness, context_recall) 측정
   - 또는 수동으로 설계 질문 10~20개를 만들고 검색 결과가 실제로 적절한지 체크

2. **섹션 청킹 프로토타입 구현**
   - `_chunk_design_by_section()` 함수를 새로 만들어 KOSHA Guide 문서 5~10개에 먼저 적용
   - 분할 기준: `^제\s*\d+\s*장` → `^\d+\.\s` → `^\d+\.\d+\s` 순서로 헤더 감지
   - 청크가 지나치게 짧으면(<200자) 다음 섹션과 병합, 2000자 초과 시 서브청킹

3. **비교 실험**
   - 동일 질문 셋으로 기존 페이지 청킹 vs 섹션 청킹 검색 결과 비교
   - context_recall(관련 문서가 몇 개나 검색됐는지)와 rerank 점수 분포 확인

4. **KGS Code 별도 처리 결정**
   - KOSHA Guide 결과가 개선되면 KGS Code도 같은 방식 적용 여부 결정
   - KGS Code는 문서 구조가 다를 수 있어 별도 파싱 필요할 수 있음

### 개선 방향 (테스트 결과에 따라)

- 개선 확인 시: `load_designs()` 내 `_chunk_pdf_by_page()` 호출을 `_chunk_design_by_section()`으로 교체
- 개선 미미 시: 현행 페이지 청킹 유지하고 section 메타데이터 정확도만 보강

## 3. laws 별표/고시 청킹 개선

### 현황

`제N조` 패턴이 없는 별표(別表) 구조는 현재 페이지 fallback으로 처리된다. 허용농도 기준표, 물질 목록 같은 테이블이 페이지 경계에서 잘린다.

### 테스트 방향

1. **별표 청킹 문제가 실제로 검색에 영향 주는지 확인**
   - 화관법 고시의 별표 내용을 직접 질문하는 케이스 몇 개 수동 테스트
   - 예: "유해화학물질 취급시설 설치 기준에서 저장시설 허용 면적은?" 같은 수치 질문
   - 검색 결과에 별표 내용이 올바르게 올라오는지 확인

2. **별표 헤더 감지 가능성 확인**
   - 화관법 고시 PDF 몇 개를 파싱해서 실제 별표 텍스트가 어떤 형태로 추출되는지 확인
   - `별표 1`, `별표\s*\d+`, `[별표 1]` 등 패턴이 일관성 있는지 체크

3. **개선 방향 결정**
   - 별표 텍스트가 파싱 후에도 구조가 있으면: 별표 헤더 기준 청킹 + 요약 텍스트 변환
   - 별표 텍스트가 파싱 후 너무 엉망이면: 해당 고시 PDF를 별도 전처리 스크립트로 정제 후 로드

### 우선순위 판단 기준

별표 내용이 실제 사용자 질문에 얼마나 자주 등장하는지에 따라 우선순위 조정. 공정위험성·법규위반 질문보다 설치기준·허용수치 질문에서 주로 필요하므로, law 검색 정확도 전체에서 별표가 차지하는 비중을 먼저 파악.
