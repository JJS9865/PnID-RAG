## 1. basics 테이블 신규 추가

기존에는 사고사례·법령·설계지침·화학물질 4개 테이블만 있었다.
화공 일반지식(교재, 논문, 보고서)을 담을 `basics` 테이블이 새로 추가되었다.

**원본 데이터**: `data/corpus/basics/{카테고리}/`

| 카테고리 | 내용 |
|----------|------|
| `기타` | 안전밸브 처리기준 등 보고서 |
| `논문_한글` | 한국어 화공 논문 |
| `전공서_영문` | 영문 화공 전공서 |

**청킹**: 페이지 단위
**주요 필드**: `title`(파일명), `chapter`("Chapter N" / "제N장" 추출)

## 3. laws — 메타데이터 대폭 추가

### 3-1. title / article 필드 추가

기존 laws 테이블에는 텍스트와 페이지 번호만 있었다.
어느 법령의 몇 번째 조항인지 알 방법이 없었다.

```
기존: text, text_vector, source, chunk_id, page, source_path
현재: + title(법령명), title_vector, article(제N조), article_vector
```

**예시**
```
title   = "산업안전보건법"
article = "제38조(안전조치)"
```

검색 결과에서 출처 표시, 필터링, 재랭킹 등에 활용 가능하다.

### 3-2. 전문(前文) 제거

법령 PDF 앞부분에는 "제1조" 이전에 법령명, 시행일, 발령기관 정보가 있다.
기존에는 이 부분도 `article=""` 청크로 저장했다.

검색에 불필요한 노이즈이므로 **제거**했다.
법령명은 `title` 필드에 이미 저장되어 있으므로 정보 손실 없다.

### 3-3. 긴 조항 서브청킹

기존에는 아무리 긴 조항이라도 통째로 1개 청크에 담았다.
조항이 수천 자인 경우 임베딩 품질이 저하된다.

**2000자 초과 시 조항 헤더를 반복 삽입하며 분할한다.**

```
[분할 전]
제15조(안전조치) ... (5000자) ...

[분할 후]
청크 1: 제15조(안전조치) ... (첫 2000자) ...
청크 2: 제15조(안전조치) [계속 2/3] ... (다음 2000자) ...
청크 3: 제15조(안전조치) [계속 3/3] ... (나머지) ...
```

헤더를 반복하므로 각 청크만 봐도 어느 조항인지 알 수 있다.

### 3-4. 빈 article 벡터 처리

조항 구조가 없는 PDF는 페이지 단위로 fallback 분할하고 `article = ""`이 된다.
빈 문자열을 그대로 임베딩하면 모델이 의미 없는 벡터를 출력해 검색 노이즈가 생긴다.

`article = ""` 인 경우 `article_vector`를 **text_vector로 복사**한다.

## 4. designs — 메타데이터 추가 + title 정제

### 4-1. title / section 필드 추가

기존 designs 테이블에는 `category`(카테고리 폴더명) 외에 메타데이터가 없었다.

```
기존: text, text_vector, source, chunk_id, page, source_path, category
현재: + title(지침명), title_vector, section(섹션번호), section_vector
```

### 4-2. title 정제 (KOSHA / KGS 분기)

기존에는 파일명을 그대로 title로 사용했다.
파일명이 URL 인코딩 형식이라 읽기 어려웠다.

`_extract_design_title()`을 신규 구현하여 파일 유형별로 분기 처리한다.

**KOSHA Guide** (파일명에 `+` 포함):
```
파일명:  P-82-2023+연속공정의+위험과+운전분석(HAZOP)+기법에+관한+기술지침
정제 후: [P-82-2023] 연속공정의 위험과 운전분석(HAZOP) 기법에 관한 기술지침
```

**KGS Code** (파일명이 `KGS`로 시작):
```
PDF 첫 페이지:
  KGS AC111 2025
  고압가스용 저장탱크 및 압력용기 제조의
  시설·기술·검사 기준

정제 후: [KGS AC111 2025] 고압가스용 저장탱크 및 압력용기 제조의 시설·기술·검사 기준
```

### 4-3. 빈 section 벡터 처리

laws의 article과 동일한 방식으로, `section = ""` 인 경우 `section_vector`를 **text_vector로 복사**한다.
